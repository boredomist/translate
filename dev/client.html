translate.client¶

Python client for translate server. Provides a simple wrapper around the HTTP interface.

class translate.client.Client(host, port=5000, scheme='http', timeout=5, **kwargs)¶

A client for interacting with translate server v1 API

Note that almost every function here may raise a translate.exceptions.TranslateException, if communication to the server fails.

batch_translate(params, ignore_timeout=False)¶

Translate multiple texts and language pairs in a single call. Returns a list of strings containing the resulting translated texts, or an Exception object, if the request failed.

This function won’t raise any exceptions explicitly, so it’s important to check the results.

can_connect()¶

Try to connect to the specified server. If a dummy request succeeds, True will be returned. If some kind of connection error (timeout, HTTP status != 200, ...) occurs, then False will be returned.

False will also be returned if the server can be reached, but does not support the API version that the Client relies on.

Parameters:

  • to_lang – Language to translate to.
  • refresh – Whether or not to ignore cached data and redownload.
info(refresh=False)¶

Collect information about this instance of the translate server.

This function uses the /api/v1/info method to collect its information.

A ServerInformation object is returned containing the relevant info.

See the API documentation for the contents of the ‘backends’ and ‘ratelimit’ dicts.

Note that if refresh=False and the data has already been collected, an additional call to /api/v1/ratelimit will have to be made to avoid stale data or guesses.

Parameters:refresh – Whether or not to ignore cached data and redownload.

Get the list of supported language pairs. If refresh is True, will ignore previously cached results and hit the server again.

languages_from(from_lang, refresh=False)¶

Generate a list of languages that the server reports being able to translate to, given the “from” language.

Return value will be a list of strings containing the possible “to” languages, or the empty list if none exist.

Parameters:

class translate.client.ServerInformation¶

This named tuple defines the various parameters of the server that the client is working with.

Members:

  • backends: { ... }
  • ratelimit: { ... } (False if ratelimit disabled)
  • sizelimit: int or False if sizelimit disabled
  • version: server version number
  • supported_api: [ API revisions server supports ]

The dict specified by the ‘ratelimit’ member will contain keys representing the reset time of the rate limit window, as well as the current ratelimit status (how many requests left per method). This information should be considered out of date as soon as the request completes, and will not update unless client.info(refresh=True) is called again. It is only stored as a convenience.

See the API documentation for information on the format of backends and ratelimit.

translate.client.exceptions¶

These are exception classes that are used by translate.client.Client. Most of these classes are simple wrappers, just to differentiate different types of errors. They can be constructed from a requests response object, or JSON ,returned from an API call.

exception translate.client.exceptions.BadLanguagePairException(lang_pair)[source]¶

Raised when the client tried to translate using a language pair not supported by the server

exception translate.client.exceptions.HTTPException[source]¶

Raised when an error occurs with the HTTP connection to the server (e.g. host is not available, doesn’t respond, etc.)

exception translate.client.exceptions.RateLimitException(limit, per, reset)[source]¶

Exception raised when a client goes over the ratelimit.

exception translate.client.exceptions.SizeLimitException(len, limit)[source]¶

Exception raised when a client tries to translate a text that is over the server’s size limit.

exception translate.client.exceptions.TranslateException[source]¶

Mostly empty base class for exceptions relating to translate.

This class is used as a catch-all for exceptions thrown by the server. If possible, a more specific subclass of this exception will be used.

classmethod from_json(obj, status_code=400)[source]¶

exception translate.client.exceptions.TranslationException[source]¶

Returned on bad parameters to /translate

exception translate.client.exceptions.TranslatorException(lang_pair, tried)[source]¶

Returned when bad parameters are passed to the /translate method. (This probably indicates some kind of API / Client bug.)
